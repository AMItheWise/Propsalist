# Security Policy

## Reporting a Vulnerability

Do not open a public issue for security vulnerabilities.

Instead:

1. Create a private report through GitHub security advisories if enabled.
2. If that is not available, contact the maintainer privately through GitHub.
3. Include reproduction details, impact, affected files, and any suggested fix.

## What to Include

- vulnerability type
- affected area or files
- reproduction steps
- expected impact
- proof of concept, if safe to share
- any known mitigations

## Response Expectations

Best effort targets:

- acknowledgement within 7 days
- triage and severity assessment after confirmation
- coordinated fix and disclosure when appropriate

## Scope Notes

Configuration mistakes in a private deployment, leaked local secrets, and
out-of-date third-party services outside this repository may be treated
separately from code vulnerabilities in this project.
